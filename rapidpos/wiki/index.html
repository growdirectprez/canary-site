Agent Architecture

Three agents, each with a defined scope and MCP tool surface. They compose — ALX orchestrates, Chirp detects, Fox manages.

ALX · Retail Ops Agent ALX The store-facing intelligence layer. Answers inventory questions, surfaces diagnostics, runs VSM tools on demand. Powered by the RapidPOS MCP Server. Chirp · Detection Engine Chirp Runs the Module Q rule catalog against live transaction data. Fires alerts to Fox when thresholds cross. No human in the loop until it chirps. Fox · Case Management Fox Receives Chirp alerts, builds case records, routes to LP team. Immutable audit trail. Closes the loop from detection to disposition. CRDM · Data Layer Centralized Retail Data Model Three-database PostgreSQL architecture: canary_app / canary_sales / canary_metrics. The structural moat. NCR doesn't have it. RapidPOS doesn't have it.
