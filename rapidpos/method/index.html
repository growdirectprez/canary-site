Canary Delivery Framework · CDF

Scaffold. Seed. Show.

From data access to a working system in 20 days. CDF is the implementation counterpart to CATz — it picks up where the assessment ends and delivers a running system, not a roadmap.

Three phases. Hard gates between each.

No phase starts until the prior gate clears. No overruns by design — if the gate doesn't open, the problem surfaces on Day 5, not Day 45.

Phase 1 Scaffold Days 1–5
  • MCP endpoint deployed and reachable
  • RapidPOS API credentials provisioned
  • Pilot store named and confirmed
  • CRDM schemas initialized
  • LOI or work order signed

Gate: ALX queries live Counterpoint data from the pilot store. If it can't read, nothing else starts.

Phase 2 Seed Days 6–15
  • Vault seeded — 200+ SKUs with taxonomy, seasonality, care data
  • First 10 Canary diagnostics running on pilot store
  • CRDM receiving daily sales and inventory feeds
  • ALX trained on retailer item taxonomy

Gate: Two VSM tools return live data. Vault has ≥ 200 entries with diagnostic annotations.

Phase 3 Show Days 16–25
  • ALX demonstrable on-site at pilot store
  • LP diagnostics running across all stores
  • Sponsor + store manager sign-off
  • Multi-store rollout plan delivered

Gate: Sponsor and store manager sign off on Phase 3 review. Rollout plan accepted.

Five workstreams. One accountable lead each.

All five run concurrently. No shared ownership on anything that blocks a gate.

WorkstreamWhat it doesAccountable
WS1 · Commercial Scope, contract, pilot store selection, rollout plan. Commercial structure determined by VAR relationship. Client Business
WS2 · API & Integration NCR Vault provisioning, Counterpoint REST API config, MCP endpoint deploy, credential handoff, endpoint testing. Client IT + RapidPOS
WS3 · Vault & Content Item master ingestion, domain taxonomy, care data, cross-sell signals. This is the moat — the Vault nobody else has built. GrowDirect + Client Biz
WS4 · CRDM & Data Three-database CRDM init, ETL from Counterpoint into canary_sales / canary_app / canary_metrics, daily feed schedule. GrowDirect + Client IT
WS5 · Platform & Infra EC2 provision, Docker stack, Cloudflare DNS, SSL. Single box. Nothing managed, nothing to maintain at launch. GrowDirect

The structural reason

CRDM is pre-built. The three-database PostgreSQL architecture that took a dev pair 12 weeks to model in 2015 exists before the engagement starts. NCR doesn't have it. RapidPOS doesn't have it. That's where the 80 days went.
