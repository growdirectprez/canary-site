VAR Quickstart

First live call in 15 minutes.

Prerequisites, credentials, and a working inventory_lookup call against your Counterpoint instance.

AudienceRapidPOS VAR engineers
Time~15 minutes
You'll needDocker · CP API key
Step1

Prerequisites

Confirm the retailer's environment and obtain credentials before continuing.

  • NCR Counterpoint 8.5.x or higher
  • RapidPOS VAR account contact vars@growdirect.io
  • Docker for local dev, or access to your deployment environment
  • Counterpoint API key with inventory read permissions
Step2

Environment

Pull the server image and configure environment variables for your retailer's tenant.

terminalbash
# Pull the server image
docker pull growdirect/rapidpos-mcp:latest

# Configure environment
export COUNTERPOINT_API_URL=https://your-cp-instance.com/api
export COUNTERPOINT_API_KEY=your_key_here
export TENANT_ID=your_retailer_id
export REDIS_URL=redis://localhost:6379

Tip. Keep COUNTERPOINT_API_KEY in your secret manager. The MCP layer never exposes it to the agent — it's used only on the server side to reach Counterpoint.

Step3

Start the server

Run the container with the environment you just exported. The server listens on port 3000 by default.

terminalbash
docker run -p 3000:3000 \
  -e COUNTERPOINT_API_URL \
  -e COUNTERPOINT_API_KEY \
  -e TENANT_ID \
  -e REDIS_URL \
  growdirect/rapidpos-mcp:latest
Server logs mcp listening :3000
CP healthcheck OK
Redis connection UP
Tools manifest 6 registered
Step4

First call

Call inventory_lookup against a known SKU at one of your stores.

requestcurl
curl -X POST http://localhost:3000/mcp \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your_mcp_api_key" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/call",
    "params": {
      "name": "inventory_lookup",
      "arguments": { "sku": "TEST-SKU", "store_ids": ["STORE1"] }
    },
    "id": 1
  }'

Expected. A JSON-RPC 2.0 envelope with a result object containing per-store stock counts and a cache_age_seconds field. If you get a 401, your bearer is wrong; a 502 means the CP credentials in COUNTERPOINT_API_KEY can't reach Counterpoint.

Step5

Connect an agent

Any MCP-compatible agent connects by pointing at http://localhost:3000/mcp with a valid API key.

claude · settings.jsonjson
{
  "mcpServers": {
    "rapidpos": {
      "url": "http://localhost:3000/mcp",
      "headers": {
        "Authorization": "Bearer your_mcp_api_key"
      }
    }
  }
}

That's it. No custom integration layer required. Claude, any Claude Code session, or any custom agent built on the MCP SDK connects natively.
